# 字体系统说明

## 用户字体目录

插件会在运行数据目录下自动创建 `fonts/`：

```text
plugin_data/astrbot_plugin_bot_menu/fonts/
```

用户把字体文件放入这个目录或其子目录即可。支持后缀：`.ttf`、`.otf`、`.ttc`、`.woff`、`.woff2`。

## 选择方式

Page 编辑器会通过后端读取 `fonts/` 目录并提供候选项。样式里的 `font_family` 只保存用户输入的字体名、生成字体族或相对路径，不保存本机绝对路径。

可填写：

- 字体文件名（例如 `MyFont`）
- 相对路径（例如 `brand/MyFont.ttf`）
- 页面候选项中的字体族
- 已安装在当前浏览器环境中的字体名

## 渲染约定

实际图片渲染时，后端会按当前菜单的 `font_family` 解析 `fonts/` 下的用户字体，并把匹配字体以内联 `@font-face` 注入渲染 HTML。这样 Page 预览和实际图片渲染读取同一套保存值。

渲染缓存指纹会包含被选中字体文件的签名；替换同名字体文件后，旧缓存不会继续命中。

## 跨环境要求

字体系统不依赖 Windows、Linux 或 macOS 的固定字体路径。运行时目录由 AstrBot 数据目录解析，字体文件只通过 `fonts/` 下的相对路径参与保存、展示和渲染。
