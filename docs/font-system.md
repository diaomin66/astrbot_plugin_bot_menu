# 字体系统说明

## 用户字体目录

插件会在运行数据目录下自动创建：

```text
plugin_data/astrbot_plugin_bot_menu/fonts/
```

支持 `.ttf`、`.otf`、`.ttc`、`.woff`、`.woff2`。

## 选择方式

Page 编辑器从后端 fonts API 读取用户字体列表。菜单只保存字体名、生成字体族或 `fonts/` 下的相对路径，不保存本机绝对路径。

## 渲染约定

菜单内所有文字都会跟随当前菜单的自定义字体，包括标题、副标题、分组标题、菜单项名称、指令文字、简介、页脚和水印。指令文字不再单独使用等宽字体。

Page 保存菜单前会等待字体加载完成，并把当前预览固化到 `render_snapshot.raster`。后端优先直接复用这份预览 PNG，因此实际图片不会再发生字体缺失、额外换行或字形回退。

旧菜单没有完整预览 PNG 时，Typst 会按同一 `fonts/` 目录解析字体做兼容渲染；要获得像素级一致性，需要重新打开 Page 并保存一次。

## 缓存

渲染缓存指纹包含被选中字体文件的签名。替换同名字体文件后，旧缓存不会继续命中。
