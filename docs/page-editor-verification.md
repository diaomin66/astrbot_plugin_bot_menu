# Page 编辑器保存与按钮验证说明

本文件记录 Page 编辑器当前的关键行为，用于后续排查“页面已修改但保存/渲染未生效”一类问题。

## 保存链路

- Page 的工作态是唯一保存源。点击保存时只会刷新当前聚焦控件，然后克隆当前完整 `menu` 快照提交到 `menus/save`，不会重新遍历弹窗里的旧控件覆盖工作态。
- 所有 Page 控件的 `input` / `change` 事件都会实时写入工作态；即使首次改成控件的 HTML 默认值（例如每行卡片改为 1 张、关闭更新时间显示），也必须立即刷新预览并标记“已修改”。
- `menus/save` 只接受 `{ "menu": ... }` 完整菜单对象；后端规范化后返回 canonical menu，Page 保存成功后以该返回值为准刷新本地状态。
- 保存成功后，后端会重新调度该菜单的渲染缓存，Page 通过 `menus/render-status/<menu_id>` 轮询缓存状态。
- 后端渲染缓存指纹覆盖完整菜单数据、渲染宽度和渲染倍率；样式字段、卡片排版和内容变化都会让旧缓存失效。

## 渲染契约

- 实际 `/menu` 图片渲染和 Page 预览共用同一个 HTML 构造器，渲染读取的字段必须来自已保存菜单文档。
- 浏览器渲染模式失败时不再静默回退到旧 Pillow 版外观；除非用户显式选择 `pillow` 模式，否则应暴露渲染错误，避免输出与 Page 预览不一致的图片。
- 远程 HTML 渲染也使用 Page 预览同形 HTML，不再维护第二套独立菜单模板。

## 按钮行为

- 删除菜单、删除分组、删除卡片、批量删除、重置样式、恢复/丢弃草稿、复制样式目标输入均使用 Page 内弹窗，不依赖浏览器原生 `confirm` / `prompt`。
- 背景编辑按钮只有在当前菜单已有背景图后显示并可切换；未上传背景图时会给出提示，不进入背景编辑模式。
- 背景图只在上传、点击“铺满背景”或点击“居中背景”时自动计算位置；保存、重载和实际渲染不得自动把 `background_image_y` 重新对齐到顶端。
- 导入菜单会先显示导入预览弹窗，确认后才调用后端 `import` 接口。

## 回归验证重点

- 顶部按钮：新建、保存、复制、删除、历史、导出、导入。
- 样式按钮：复制样式到其他菜单、一键修复颜色、一键重置样式、背景上传/编辑。
- 内容按钮：分组复制/删除，卡片新增/编辑/复制/删除。
- 批量按钮：批量模式、全选、清除、启用、禁用、排版、复制、删除、上下移动。
- 后端：`MenuRenderCache.fingerprint` 必须随所有 `DEFAULT_STYLE` 字段变化而变化；保存后必须调度缓存刷新；包含背景偏移、卡片字体、间距、水印的完整菜单快照保存后必须能在 `build_preview_html` / 实际渲染 HTML 中读取。
