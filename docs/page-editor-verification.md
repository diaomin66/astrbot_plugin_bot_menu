# Page 编辑器保存与按钮验证说明

本文件记录 Page 编辑器当前的关键行为，用于后续排查“页面已修改但保存/渲染未生效”一类问题。

## 保存链路

- 在弹窗编辑器打开时，按 `Ctrl+S` 会先刷新当前弹窗内的 `input` / `change` 状态，再构造保存 payload。
- 所有 Page 控件的 `input` / `change` 事件都会重新同步状态；即使首次改成控件的 HTML 默认值（例如每行卡片改为 1 张、关闭更新时间显示），也必须立即刷新预览并标记“已修改”。
- 保存成功后，后端会重新调度该菜单的渲染缓存，Page 通过 `menus/render-status/<menu_id>` 轮询缓存状态。
- 后端渲染缓存指纹覆盖完整菜单数据、渲染宽度和渲染倍率；样式字段、卡片排版和内容变化都会让旧缓存失效。

## 按钮行为

- 删除菜单、删除分组、删除卡片、批量删除、重置样式、恢复/丢弃草稿、复制样式目标输入均使用 Page 内弹窗，不依赖浏览器原生 `confirm` / `prompt`。
- 背景编辑按钮只有在当前菜单已有背景图后显示并可切换；未上传背景图时会给出提示，不进入背景编辑模式。
- 导入菜单会先显示导入预览弹窗，确认后才调用后端 `import` 接口。

## 回归验证重点

- 顶部按钮：新建、保存、复制、删除、历史、导出、导入。
- 样式按钮：复制样式到其他菜单、一键修复颜色、一键重置样式、背景上传/编辑。
- 内容按钮：分组复制/删除，卡片新增/编辑/复制/删除。
- 批量按钮：批量模式、全选、清除、启用、禁用、排版、复制、删除、上下移动。
- 后端：`MenuRenderCache.fingerprint` 必须随所有 `DEFAULT_STYLE` 字段变化而变化；保存后必须调度缓存刷新。
