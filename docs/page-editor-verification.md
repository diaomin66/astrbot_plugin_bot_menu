# Page 编辑器保存与按钮验证说明

本文件记录 Page 编辑器当前关键行为，用于排查“页面已修改但保存或渲染未生效”的问题。

## 保存链路

- Page 的工作态是唯一保存源；保存时只刷新当前聚焦控件，再克隆完整 `menu` 快照提交到 `menus/save`。
- 所有控件的 `input` / `change` 事件必须实时写入工作态，并刷新预览、标记已修改。
- `menus/save` 只接受 `{ "menu": ... }` 完整菜单对象；后端规范化后返回 canonical menu，Page 以返回值刷新本地状态。
- 保存成功后，后端会优先把 `render_snapshot.raster` 直接写入 Typst 渲染缓存；没有完整预览 PNG 时才调度 Typst 兼容编译。
- Page 通过 `menus/render-status/<menu_id>` 轮询缓存状态。

## 渲染契约

- 图片渲染只有 Typst 一条路。
- 最高保真路径是 Page 保存的完整预览 PNG：后端直接复用该 PNG，不重新排版文字、不重新计算布局、不重采样。
- 旧菜单缺少完整预览 PNG 时，Typst 使用保存的几何、文本、颜色、图片和字体信息做兼容输出；像素级一致性要求重新打开 Page 保存。
- 后端缓存指纹必须覆盖完整菜单数据、渲染宽度、渲染倍率和选中字体签名。

## 按钮行为

- 删除菜单、删除分组、删除卡片、批量删除、重置样式、恢复/丢弃草稿、复制样式目标输入均使用 Page 内弹窗。
- 背景编辑按钮只有在当前菜单已有背景图后显示并可切换；未上传背景图时只提示，不进入背景编辑模式。
- 背景图只在上传、点击“铺满背景”或点击“居中背景”时自动计算位置；保存、重载和实际渲染不得自动把位置重新对齐到顶端。
- 背景拖动/拉伸期间不得替换 `menu.style` 对象；位置和尺寸必须始终写入当前工作态。
- 导入菜单会先显示导入预览弹窗，确认后才调用后端 `import` 接口。

## 回归验证重点

- 顶部按钮：新建、保存、复制、删除、历史、导出、导入。
- 样式按钮：复制样式到其他菜单、一键修复颜色、一键重置样式、背景上传/编辑。
- 内容按钮：分组复制/删除，卡片新增/编辑/复制/删除。
- 批量按钮：批量模式、全选、清除、启用、禁用、排版、复制、删除、上下移动。
- 后端：保存后必须能命中 Typst 快路径；没有快照时必须进入 Typst 兼容编译并给出明确状态。
